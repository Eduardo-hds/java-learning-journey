# Chapter 6 — Constants

## Overview

Not every value in a program should be allowed to change.

Imagine a program where the value of π (`3.14159`), the speed of light, or the number of days in a week could accidentally be modified. Bugs like these can be difficult to find and may produce incorrect results throughout the application.

Java solves this problem with **constants**.

In this chapter, you'll learn what constants are, why they exist, how the `final` keyword works, and the conventions professional Java developers follow when declaring constant values.

---

# Why Do Constants Exist?

Variables are designed to change.

Example:

```java
int lives = 3;

lives = 2;
lives = 1;
```

This makes sense because the number of lives in a game changes over time.

But some values should **never** change.

Examples:

* Number of months in a year
* Number of hours in a day
* Value of π
* Tax rates loaded at startup
* Company name
* Maximum number of login attempts

For these values, Java provides **constants**.

---

# What Is a Constant?

A **constant** is a variable whose value can be assigned **only once**.

In Java, constants are declared using the `final` keyword.

General syntax:

```java
final dataType CONSTANT_NAME = value;
```

Example:

```java
final double PI = 3.141592653589793;
```

After initialization, `PI` cannot receive another value.

---

# The `final` Keyword

The keyword `final` tells the compiler:

> "This variable may be assigned only once."

Example:

```java
final int DAYS_IN_WEEK = 7;
```

Trying to change it:

```java
DAYS_IN_WEEK = 8;
```

Results in a compilation error.

This error is caught before the program even runs, helping prevent bugs.

---

# Variables vs Constants

Compare the two:

```java
int score = 100;
```

The value can change:

```java
score = 200;
```

Now compare with:

```java
final int MAX_PLAYERS = 4;
```

Attempting:

```java
MAX_PLAYERS = 8;
```

Compilation error.

---

# Initialization Rules

A `final` variable must receive a value exactly once.

You can initialize it immediately:

```java
final int MAX_SCORE = 100;
```

Or later:

```java
final int MAX_SCORE;

MAX_SCORE = 100;
```

Both are valid, as long as the assignment happens only once.

---

# Why Are Constants Useful?

## 1. Preventing Accidental Changes

Imagine:

```java
final double PI = 3.141592653589793;
```

Nobody can accidentally write:

```java
PI = 5;
```

The compiler prevents it.

---

## 2. Improving Readability

Compare:

```java
double area = radius * radius * 3.141592653589793;
```

With:

```java
final double PI = 3.141592653589793;

double area = radius * radius * PI;
```

The second version is easier to understand.

---

## 3. Easier Maintenance

Suppose a tax rate changes.

Bad approach:

```java
price * 0.18
```

Used in 30 different places.

Better:

```java
final double TAX_RATE = 0.18;
```

Now only one value needs to be updated.

---

# Naming Conventions

Java follows a specific convention for constants.

Variables use:

```java
customerAge
```

Constants use:

```java
CUSTOMER_AGE
```

General rules:

* All uppercase letters
* Words separated by underscores

Examples:

```java
MAX_SPEED
MIN_PASSWORD_LENGTH
DEFAULT_TIMEOUT
PI
SECONDS_PER_DAY
```

This makes constants immediately recognizable in code.

---

# Where Should Constants Be Declared?

For now, declare constants inside the `main()` method:

```java
public static void main(String[] args) {

    final int DAYS_IN_WEEK = 7;

}
```

In larger applications, constants are often declared at the class level or in dedicated configuration classes. You'll learn these techniques in later modules.

---

# Compile-Time Constants

Some constants are known entirely at compile time.

Example:

```java
final int DAYS = 7;
```

The compiler may optimize these values directly into the generated bytecode.

This optimization is one reason constants can improve performance slightly, although readability and safety are the primary benefits.

---

# Common Beginner Mistakes

### Forgetting `final`

Incorrect:

```java
double PI = 3.141592653589793;
```

This creates a normal variable, not a constant.

Correct:

```java
final double PI = 3.141592653589793;
```

---

### Using camelCase

Incorrect:

```java
final int maxPlayers = 4;
```

Correct:

```java
final int MAX_PLAYERS = 4;
```

Following naming conventions makes code easier for other developers to read.

---

### Trying to Reassign a Constant

Incorrect:

```java
final int MAX_USERS = 100;

MAX_USERS = 200;
```

Compilation error.

---

# Best Practices

* Use `final` whenever a value should never change.
* Name constants using uppercase letters with underscores.
* Replace "magic numbers" with named constants.
* Keep constants close to where they are used, unless they are shared across multiple classes.

---

# Magic Numbers

A **magic number** is a literal value used directly in code without explanation.

Example:

```java
double area = radius * radius * 3.141592653589793;
```

What is `3.141592653589793`?

A reader must guess.

Better:

```java
final double PI = 3.141592653589793;

double area = radius * radius * PI;
```

The code is now self-explanatory.

Avoiding magic numbers is a hallmark of clean, professional code.
