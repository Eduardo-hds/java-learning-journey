# Chapter 5 — Getting to Know IntelliJ IDEA

## Overview

Writing Java code directly in a text editor is possible, but in professional development it is not efficient.

In real-world projects, developers use an Integrated Development Environment (IDE).

In Java development, one of the most powerful and widely used IDEs is IntelliJ IDEA.

This chapter introduces IntelliJ IDEA, its main features, and how to use it to create and manage Java projects effectively.

## Objectives

By the end of this chapter, I will understand:

- Why IntelliJ IDEA is widely used in Java development

- How to create a Java project in IntelliJ

- How to run Java applications
- How to use basic debugging tools
- How to navigate the IDE efficiently

## What is an IDE?

An IDE (Integrated Development Environment) is a tool that combines:

- Code editor
- Compiler integration
- Debugger
- Project manager
- Build tools integration

Instead of using multiple tools separately, an IDE centralizes everything in one place.

---

## Why IntelliJ IDEA?

IntelliJ IDEA is one of the most popular Java IDEs because it provides:

- Intelligent code completion
- Automatic error detection
- Refactoring tools
- Built-in debugger
- Integration with build tools (Maven, Gradle)
- Excellent performance and usability

It is widely used in professional Java development.

---

## Editions of IntelliJ IDEA

### Community Edition

- Free
- Open-source
- Supports core Java development
- Enough for learning and most projects

### Ultimate Edition

- Paid
- Includes advanced enterprise features
- Web development tools
- Framework support (Spring, Jakarta EE, etc.)

For this Bootcamp, Community Edition is sufficient.

---

## Creating a Java Project

When creating a project in IntelliJ IDEA:

1. Select "New Project"
2. Choose Java
3. Select installed JDK
4. Define project name
5. Choose location

IntelliJ will automatically generate a project structure.

---

## Project Structure in IntelliJ

A typical Java project contains:

- src/ → source code folder
- .idea/ → IDE configuration
- out/ → compiled output files

Inside src, you organize code using packages.

---

## Running a Java Program

To run a Java program:

- Create a class with a main method
- Click the Run button (green triangle)
- Or use shortcut keys

IntelliJ automatically:
- Compiles the code
- Executes the bytecode using JVM
- Shows output in the console

---

## Debugging in IntelliJ

IntelliJ provides a powerful debugger.

Main features:

- Breakpoints
- Step over (execute line by line)
- Step into (enter method execution)
- Step out (exit method)
- Variable inspection
- Call stack analysis

Debugging helps understand program execution deeply.

---

## Navigation Features

Important navigation tools:

- Search everywhere (Shift twice)
- Go to class
- Go to file
- Go to symbol
- Recent files

These tools improve productivity significantly.

---

## Refactoring Tools

IntelliJ allows safe code changes:

- Rename variables/classes
- Extract methods
- Move classes/packages
- Optimize imports

Refactoring ensures clean and maintainable code.

---

## Best Practices

- Always organize code into packages
- Keep project structure clean
- Use meaningful names for classes and methods
- Use built-in IDE suggestions instead of manual fixes
- Learn shortcuts early to improve productivity

---

## Common Mistakes

- Writing all code in the default package
- Ignoring warnings from the IDE
- Not using debugging tools
- Not organizing project structure properly
